**:fire: Управление видимостью и положением столбцов в таблицах**

**:fire: Отключение возможности создания публичных ссылок в проекте**

В [настройках проекта](/docs/settings) добавили новую вкладку **Функциональность** и соответствующую опцию:

![2026.01_-_отключение_публикации_файлов.png](/release_notes/2026.01_-_отключение_публикации_файлов.png)

**:sparkles: Доработки функционала**
- Добавили быстрые фильтры по статусам согласования в [документах](/docs/folders).
- Во всплывающей подсказке об этапе [согласования](/docs/reviews) добавили информацию о пользователе, который завершил этап.
- Добавили обязательное поле **Название проекта** при формировании [пакета паредачи](/docs/documentsTransfers). По умолчанию значение заполняется из названия текущего проекта, но можно его изменить.
- Добавили отключение возможность публикации файлов в [настройках проекта](/docs/settings) в новой вкладке **Функциональность.**

**:hammer_and_wrench: Исправление ошибок**
- Исключили возможность отключения сервера из-за высоких нагрузок, вызываемых некоторыми запросами пользователей.
- Убрали возможность выбрать недоступные для создания типы замечаний.
- Исправили ошибку, при которой в отчёт не экспортировались согласования в статусе "Черновик" после перезапуска.
- Поправили небольшие неточности интерфейса замечаний и согласований

---
<span style="background: linear-gradient(45deg, #7AA6CF, #7D96AD); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">API</span>

В [публичный API](/general/api) добавили:
- методы для получения журнала событий из HUB (см. контроллер `UserLog`)
- возвращаемое поле `code` (код проекта) в методах `GET /public/v1/projects/{projectId}` и `GET /public/v1/projects`
- метод `PATCH /public/v1/reviews` для обновления основных полей комплекта согласования
- метод `POST /public/v1/reviews/versions` для получения итерации согласования, в т. ч. рекурсивного получения предыдущих версий
- возвращаемое поле `topVersion.stages.completedBy` (кто завершил этап согласования) в методе `POST /public/v1/reviews`
- методы для работы с пакетами передачи (получение сертификата для скачивания файлов, получение правил доступа к пакету, получение файлов) (см. контроллеры `AcceptanceCertificate`, `DocumentsTransferAccesses`, `DocumentsTransferItem`)
