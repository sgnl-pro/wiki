Новый модуль SIGNAL для ведения исполнительной (и не только) документации в цифровом виде.
Подробнее см. здесь: https://wiki.sgnl.pro/forms

---
<span style="background: linear-gradient(45deg, #2D4E67, #112538); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">HUB</span>

**:sparkles: Доработки функционала**
- Реализовали сохранение информации о назначенных пользователям лицензий перед тем, как они закончатся. Информация хранится 2 недели и при продлении лицензий позволяет восстановить их нужным пользователям за один клик.
- Добавили у проекта галочку "Проект по умолчанию для новых пользователей".
- Включили возможность редактирования данных пользователя администраторами, если пользователь добавлен только в одну компанию.
- При выборе ролей в таблице пользователей отмеченные роли теперь отображаюся вверху списка.
- Добавили в профиль пользователя указание его Telegram.
- Обновили текст приветственного письма при добавлении в компанию и проект.

---

<span style="background: linear-gradient(45deg, #00D1FF, #0695D7); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">DOCS</span>

**:fire: Новый BIM-вьювер**

Реализовали собственный просмотрщик BIM-моделей в SIGNAL. Сейчас он доступен только для форматов GLB и IFC, в следующих релизах появится поддержка форматов Revit и Navisworks.

![2026-12-сводная.jpg](/release_notes/2026-12-сводная.jpg =1200x)

**:fire: ИИ-ассистент**

> Функционал доступен при наличии лицензии, обращайтесь в поддержку.
{.is-info}

Реализовали ИИ-ассистента, который помогает создавать замечания и текстовые файлы по проекту, формулировать комментарии, наводить порядок в файлах и папках, а также может подсказать по функционалу системы.

![ai.png](/release_notes/ai.png =1200x)

**:fire: Генерация файлов предписаний из замечаний**

Реализовали возможность генерировать файлы DOCX с данными из замечания. Для этого к типу замечания в настройках проекта нужно добавить [шаблон с нужным оформлением и ключами-свойствами замечания](/docs/settings#temp_doc). При наличии такого шаблона в карточке замечания появляется кнопка генерации:

![2026-12-предписание.png](/release_notes/2026-12-предписание_.png)  

:sparkles: Включили возможность пользователю с доступом на Создание согласования изменять тип при перезапуске:

![2026-12-изменение_типа_согласования.png](/release_notes/2026-12-изменение_типа_согласования.png =600x)
