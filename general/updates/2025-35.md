**:fire: Плиточный режим просмотра документов**

Реализовали плиточный режим просмотра в [документах](/docs/folders). Превью создается для  PDF, CAD моделей, изображений и видеофайлов при первом открытии документа любым из пользователей. У некоторых легких файлов до определенного размера превью будет создаваться при загрузке их в DOCS. Для CAD моделей доступна кнопка **Обновить превью** на панели инструментов.

![превью_в_папке.png](/release_notes/превью_в_папке.png =1200x)

**:fire: Функция печати из PDF**

Добавили функцию печати при [просмотре PDF](/docs/viewers#pdf). Она позволяет открыть файл в просмотрщике вашего браузера как для последующей печати (в том числе отдельных листов), так и в качестве альтернативного просмотра (например перелистывание документа скроллом).

![функция_печати_из_pdf.png](/функция_печати_из_pdf.png =1200x)

**:sparkles: Доработки функционала**
- Добавили отображение количества файлов и папок: общее и выбранных.
- Добавили индексацию пометок в рамках одного [замечания](/docs/issues) (чтобы в комментарии к замечанию можно было сослаться на конкретную пометку).
- Улучшили создание текстовых пометок:
	- при создании сразу включается режим редактирования (и без TYPE TEXT)
	- добавили настройку стилей и размеров текста
	- устранили ошибку со скроллом в процессе переноса на новую строку
	- добавили подсветку поля прозрачным цветом для пустой пометки
	- добавили выход из редактирования по <kbd>Esc</kbd>
	- добавили выделение всего текста при переход в редактирование (двойной клик)
  ![photo_2025-11-14_10-41-39.jpg](/release_notes/photo_2025-11-14_10-41-39.jpg =1200x)
- Добавили удаление пометок кнопкой <kbd>Delete</kbd> и сброс выделения пометки кнопкой <kbd>Esc</kbd>.
- Добавили предупреждение о сохранении пометок при переключении между файлами.
- Доработали дисклеймер при [простом просмотре](/docs/sharedSettings#publicmodels): сместили вбок и убрали возможность его закрыть.

**:hammer_and_wrench: Исправление ошибок**
- Поправили отображение лупы при поиске в браузере **Firefox.**

**:sparkles: Доработки функционала**
- В настройках [41 карточки](/dash/cards-40#card-41): 
	- добавили поле-список **Даты для расчета** для выбора Базового плана, даты из которого будут отображаться в столбце **ПЛАН**
 	- убрали поле **Общий объем,** который теперь будет высчитывается из планового объема в MPP
- В [50 карточке](/dash/cards-50#card-50) в списке с выбором дат добавили функцию **Диапазон,** с помощью которой удобно выбирать диапазон дат.
- Доработали [67 карточку](/dash/cards-60#card-67):
	- если комплект **Согласован с замечаниями,** то статус бэйджика на 4-ом уровне будет отображаться желтым цветом. Если комплект **На проверке,** то статус бэйджика на 4-ом уровне будет отображаться синим цветом
 	- поправили некорретное отображение даты согласования, если согласование находится в работе, черновик или отменено: **Согласовано в срок Invalid Date**
	- на 3 и 4 уровне добавили количество активных замечаний на комплекте (Открыто, В работе, На проверке, Отклонено, Не принято)
