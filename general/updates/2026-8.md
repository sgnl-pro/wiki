<span style="background: linear-gradient(45deg, #00D1FF, #0695D7); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">DOCS</span>

**:fire: Лента с предпросмотром всех файлов**

Вместо стрелок для пролистывания PDF файлов в папке теперь доступна полноценная лента для [предпросмотра и переключения между всеми файлами](/docs/viewers) любых форматов.

Кнопка открытия ленты находится в правом нижнем углу экрана. При свернутой ленте на этой кнопке также доступны стрелки для переключения между файлами.

При открытии файла из [карточки замечания](/docs/issues), [комплекта согласования](/docs/reviews) или [передачи](/docs/documentsTransfers) также пролистываются все соседние файлы.

![лента_превью_файлов.png](/release_notes/лента_превью_файлов.png =1200x)

**:fire: Сборки по общим координатам и внутренним началам**

При создании и редактировании [сборки](/docs/viewers#assemblies) теперь можно выбрать вариант размещения одной модели относительно предыдущей, чтобы не вводить координаты вручную. Доступен выбор из трёх вариантов: по общим координатам, совмещению начал и совемещению геометрических центров.

![вариант_размещения_модели_в_сборке.png](/release_notes/вариант_размещения_модели_в_сборке.png =1200x)

**:sparkles: Доработки функционала**
- В браузере модели для файлов Navisworks при [просмотре в Forge](/docs/viewers#bim) добавили поиск по Id элемента.
- В сборках добавили кнопку `Вернуться к общему виду`, которая отображается при переключении на другие виды моделей.

**:hammer_and_wrench: Исправление ошибок**
- Исправили ошибку, при которой в документах отображались не все атрибуты в фильтрах.
- Поправили калибровку и устранили ещё несколько небольших ошибок линейки PDF в мобильной версии.
- В Forge вместо идентификаторов в обозревателе документов теперь отображаются имена файлов.
- Исправили ошибку, при которой закрывалась сборка при добавлении вложения в замечание.
- Поправили форматирование отчёта замечаний в PDF при длинном описании, а также формирование отчёта с большим количеством замечаний.
- Исправили время отправки уведомлений об истечении срока согласований при завершении предыдущего этапа до 13:00.
- Исправили ошибку, при которой некоторые согласования отображались в статусе "Черновик" после перезапуска.

**:fire: Импорт данных из JSON 1С в карточке "22_Освоение денег по месяцам"**

Для импорта данных в [карточке 22](/dash/cards-20#card-22) требуется в настройках карточки указать папку, в которой лежат выгруженные из 1С файлы формата json или txt, и выбрать компании, по которым собрать информацию. При импорте данных из 1С для редактирования будет доступен только столбец ПЛАН.

![2026.8_-_импорт_в_22_карточке.png](/2026.8_-_импорт_в_22_карточке.png =1200x)

**:sparkles: Доработки функционала**
- В [карточке 15](/dash/cards-10#card-15) при наведени на область теперь выводим значение области за соответствующий месяц.
- В [карточке 41](/dash/cards-40#card-41) добавили плановый процент выполнения.

**:hammer_and_wrench: Исправление ошибок**
- Поправили, чтобы карточка 15 не ломалась из-за нового поля `Тип` в Google-таблице.
- Исправили сброс дат при изменений фильтро в [карточках 50 и 51](/dash/cards-50).
- Поправили привязку в карточках 50 и 51, чтобы учитывался тип.
- Сделали, чтобы в нужном случае выводился статус `Согласовано с замечаниями` вместо `Согласовано` в [карточках 65](/dash/cards-60#card-65).
- Исправили фильтрацию в [карточке 66](/dash/cards-60#card-66) на дату завершения согласования, вместо даты начала.

**TOOLS FOR REVIT 2.18.7.0**

:sparkles: Добавили новый сценарий обработки моделей в инструменте [Серверная обработка](/tools/revit/attachments). Режим `Сохранить в DOCS` позволяет выгрузить модели из проводника или Revit сервера напрямую, без их открытия на рабочей машине. При включении данного режима остальные сценарии обработки становятся недоступны.

![tools_-_сохранить_в_docs.png](/release_notes/tools_-_сохранить_в_docs.png =1200x)

**TOOLS FOR NAVIS 2.22.2.0**

:sparkles: В инструмент «Экспорт в IFC» добавили:
- новый вариант экспорта `Все видимые` — в данном случае в IFC будут экспортированы все элементы модели, которые являются видимыми на текущем виде
![tools_-_все_видимые.png](/release_notes/tools_-_все_видимые.png =600x)
- прогресс-бар выполнения экспорта
![tools_-_ifc_обработка.jpg](/release_notes/tools_-_ifc_обработка.jpg)
- экспорт цветов в IFC, если модель раскрашена инструментами TOOLS. Значение цвета берется из свойства `SIGNAL - UserColor`
![tools_-_ifc_материал_-_цвет.png](/release_notes/tools_-_ifc_материал_-_цвет.png =900x)
