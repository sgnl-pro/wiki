**:fire: Перезапуск согласований по трём сценариям**

Реализовали три сценария перезапуска согласования:
1. **Полный перезапуск** — когда нужно аннулировать все файлы из предыдущей итерации согласования, независимо от наличия в них новых версий
2. **Запуск изменений** — когда нужно согласовать измененные файлы, чтобы их предыдущие версии аннулировались, а неизмененные файлы не потеряли свою силу. В зависимости от настроек типа согласования предыдущие версии аннулируются либо в начале, либо в конце новой итерации
3. **Дополнительное согласование** — когда нужно согласовать дополнительный файл, но чтобы никакие файлы из предыдущей итерации не потеряли свою силу, либо когда нужно получить альтернативную версию комплекта без отмены старой

**:sparkles: Доработки функционала**
- Добавили возможность выделять элементы с помощью <kbd>Shift</kbd>.
- Оптимизировали открытие больших PDF.
- Вывели комментарий актуальной версии файла при просмотре в папке.
- Реализовали поиск по названию папок в документах.
- Добавили дату создания версии при просмотре в Forge.
- Добавили кнопку `Свернуть все` в браузере модели в Forge.
- Внесли ряд доработок в новый график Ганта.
- Сделали так, чтобы высота шрифта и выравнивание текста сохранялись при создании следующей пометки.
- Добавили возможность отображать графические вложения при экспорте замечаний в PDF.
- Добавили фильтрацию по дате устранения в замечаниях.
- Оптимизировали открытие календаря согласований.
- Добавили в реестр согласований быстрый фильтр `Отправлено мной`.
- Добавили возможность на финальном этапе согласования выбрать роль или пользователя для дополнительного уведомления
- Добавили кнопку `Открыть передачу` в окно добавления файлов в пакет.
- Вывели подсказки с выбранными папками для `Отстроить структуру` в настройках согласований.
