**:sparkles: Доработки функционала**
- Реализовали проверку и предупреждение при удалении роли, если она используется в типах замечаний или согласований.
- Добавили отдельную форму пригласительного письма для пользователей с SSO.


---

<span style="background: linear-gradient(45deg, #00D1FF, #0695D7); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">DOCS</span>

**:fire: Онлайн-редактор офисных документов**

С файлами форматов DOCX, DOC, XLSX, XLSM, XLS, PPTX, PPT теперь можно работать в полноценном офисном редакторе в режиме онлайн совместно с коллегами. При сохранении после редактирования создаётся версия документа. Доступно комментирование под учётной записью SIGNAL. Сейчас поддерживаются файлы размером до 200 Мб.

![офисный_редактор.png](/release_notes/офисный_редактор.png)

**:fire: Просмотрщик облаков точек**

Также теперь доступен просмотр облаков точек. Сейчас поддерживаются файлы размером до 5 Гб.

![облака_точек.jpg](/release_notes/облака_точек.jpg =1200x)

**:sparkles: Доработки функционала**
- В сборки добавили версии файлов с возможностью перехода к ним и предупреждение, если используются неактуальные версии. В папке такая сборка будет подсвечена жёлтым цветом. Также в замечаниях в сборках теперь доступны функции прикрепления элементов и пометки:
![пометки_в_сборках.jpg](/release_notes/пометки_в_сборках.jpg =1200x)
- Для редакторов согласований добавили возможность менять плановую дату этапа из комплекта. Также добавили соответствующую опцию в настройки типа согласования, по умолчанию эта возможность отключена:
![изменение_плановой_даты.png](/release_notes/изменение_плановой_даты.png =900x)
- В настройки типа согласования также добавили поле `Наблюдатели` (пользователи и/или роли, которым должны приходить уведомления о согласованиях этого типа) и вариант ответственного за этап "Назначить в процессе" — в этом случае ответственного выбирает пользователь, заверщающий предыдущий этап:
![назначить_в_процессе.png](/release_notes/назначить_в_процессе.png =900x)
- В комплект согласования при наведении на этап добавили список пользователей, за которыми закреплена роль (если в качестве ответственного в настройках выбрана роль, а не пользователь).
- Добавили поле `Комментарий` при запуске согласования из документов.
- Добавили дату загрузки последней версии файла при перезапуске согласования.

**:hammer_and_wrench: Исправление ошибок**
- Исправили некорректное отображение некоторых шрифтов при серверной обработке PDF.
- Вернули возможность автору замечания менять его тип, выбирая среди доступных.
- Вывели в документах корректную дату аннулирования файла.
- Исправили некорректное отображение сроков в календаре согласований в случае, когда согласование этапа произошло после 13:00.
- Добавили контекстное меню папки в дереве через нажатие правой кноки мыши.

<span style="background: linear-gradient(45deg, #A09EFF, #4B47FF); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">DASHBOARD</span>

**:fire: Расшаривание строительных моделей по ссылке для внешних систем**

Реализовали возможность встраивания содержимого карточек во внешние системы, например корпоративных порталах компаний. Ссылку можно обновлять по расписанию.

![публичная_ссылка_dash_1.jpg](/release_notes/публичная_ссылка_dash_1.jpg)
![публичная_ссылка_dash_2.jpg](/release_notes/публичная_ссылка_dash_2.jpg)

**:sparkles: Доработки функционала**
- Перенесли кнопку настройки доступов выше. Кнопка синхронизации карточек доски и группы теперь появляется, если на доске или в группе есть карточки с доступом на заполнение. Доступы к операциям больше не наследуют права от верхнего уровня.
- Добавили возможность не скрывать, а деактивировать карточки:
...
- Добавили сворачивание групп карточек.
- При нажатии на кнопку "Обновить" теперь обвновляются даты карточек "41_План-факт по объемам", "41_План-факт по объемам MPP" и "46_План-факт по объемам с прогнозам".
- Добавили открытие реестра замечаний DOCS при нажатии на дату в карточках 61 и 62.

<span style="background: linear-gradient(45deg, #7AA6CF, #7D96AD); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">API</span>

В [публичный API](/general/api) добавили:
- метод `/public/v1/items/versions/update` для добавления комментария к версии файла
- метод `/public/v1/items/item-version`, который по `itemVersionId` возвращает `itemId` и `versionId`
- контроллер `IssueComment` с методами для получения и создания комментариев в журналах замечаний
- поле `itemVersionIds` в `Issues`
