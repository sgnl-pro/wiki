**:fire: Настройки уведомлений в профиле пользователя**

В разделе [Уведомления в профиле HUB](/hub/user#notifications) появились расширенные настройки. Как и ранее, можно выбрать, куда вы хотите их получать (сейчас временно доступна только почта). Также в общих настройках можно настроить объединение уведомлений в одно письмо, произошедших за выбранный период. Если объединение выключено, то каждое уведомление будет приходить отдельно:

![2026-9-настройки_уведомлений-общие.png](/release_notes/2026-9-настройки_уведомлений-общие.png =600x)

Далее по модулям можно включить / отключить и настроить объединение для уведомлений каждого типа, которые сейчас доступны в системе. Этот список будет расширяться:

![2026-9-настройки_уведомлений-docs.png](/release_notes/2026-9-настройки_уведомлений-docs.png =1200x)

В настройках DOCS осталась возможность выключить уведомления для конкретного проекта и управлять списком папок, по которым вы хотите получать уведомления:

![2026-9-настройки_уведомлений-проект.png](/release_notes/2026-9-настройки_уведомлений-проект.png =900x)

Подробнее о настройках уведомлений см. в [статье](/hub/user#notifications).

**:fire: Экспорт протокола по комплекту согласования**

В [комплекте согласования](/docs/reviews#set_card) добавили кнопку Экспорт отчета, которая позволяет настроить и выгрузить в PDF протокол согласования. Можно выбрать либо все итерации, либо только текущую:

![2026-9-отчет_о_согласовании.png](/release_notes/2026-9-отчет_о_согласовании.png =1200x)

![2026-9-отчет_о_согласовании_результат.png](/release_notes/2026-9-отчет_о_согласовании_результат.png =1200x)

**:fire: Настройка дополнительного текста в акте передач**

В настройках [типа перерадчи](/docs/documentsTransfers) добавили необязательное поле "Дополнительный текст в акте", при заполнении которого он будет добавляться в конец акта при генерации:

![2026-9-настройки_акта.png](/release_notes/2026-9-настройки_акта.png =1200x)

![2026-9-акт_с_доп_текстом.png](/release_notes/2026-9-акт_с_доп_текстом.png =600x)

**:sparkles: Доработки функционала**
- Добавили отображение активных пользователей во время просмотра [офисных документов](/docs/viewers#office) (ранее отображались только при редактировании).
- Расширили отображаемый путь до папки и файла на всё свободное место.
- Добавили текст о возможности вставлять изображения из буфера обмена при добавлении вложения к [замечанию](/docs/issues). Сама возможность была и ранее.
- Добавили статусы документов при добавлении их в комплект согласования.
- Добавили автоматическую подстановку шифра и названия при создании [пакета передачи](/docs/documentsTransfers) из комплекта согласования. При необходимости значения можно менять.
- Добавили возможность менять порядок значений списочного атрибута в настройках проекта.

**:hammer_and_wrench: Исправление ошибок**
- Поправили отображение ролей в комментариях в отчёте по замечаниям.
- Исправили ошибку, при которой не отменялось и не штамповалось согласование в некоторых случаях.
- Исправили ошибку при формировании акта передачи с большим количеством документов.

**:fire: Привязка XLSX из DOCS**

Обновлять данные карточек теперь можно не только через Google Таблицы, но и через XLSX файлы в DOCS. Данные можно редактировать онлайн.

![2026-9-привязка_xlsx.png](/release_notes/2026-9-привязка_xlsx.png =1200x)

**:fire: Карточка «62_Замечания DOCS» в «Мой отчет»**

[Карточка](/dash/cards-60#card-62) доступна каждому пользователю в разделе «Мой отчет». Отображает все замечания пользователя по всем проектам, в которых он состоит. Есть возможность отфильтровать по проекту, типу и статусу замечания:

![2026-9-карточка_62.png](/release_notes/2026-9-карточка_62.png =1200x)

**:fire: Карточка «03_SLA по ролям» в «Отчет компании»**

Похожа на карточку [66_SLA по ролям](/dash/cards-60#card-66). Считает средние SLA по всем проектам и процент согласования в срок сотрудников:

![2026-9-карточка_03.png](/release_notes/2026-9-карточка_03.png)

**:sparkles: Доработки функционала**
- Добавили комментарий при публикации еженедельного отчета.
- В [карточка 14](/dash/cards-10#card-14) сделали обрезку пустого значение на графике в факте.
- В [карточке 65_2](/dash/cards-60#card-65) добавили фильтры с датой.
