<span style="background: linear-gradient(45deg, #00D1FF, #0695D7); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">DOCS</span>

**:sparkles: Доработки функционала**
- В окно [загрузки файлов](/docs/folders#uploadings) (которое появляется при наличии у папки масок) добавили поле с получателями уведомления о загрузке и поле для ввода комментария к версии файла. Значения можно заполнять сразу для нескольких выбранных файлов. Также вывели в список файлов информацию о заполнении этих полей и обновляемые версии существующих в папке файлов:
![2026-13_-_окно_загрузки.png](/release_notes/2026-13_-_окно_загрузки.png =1200x)
- Сняли ограничение с кнопки `Обновить версии` при выделении файлов в [комплекте согласования](/docs/reviews#set_files): теперь все выделенные неактуальные файлы будут обновлены, а актуальные пропущены.
- В настройках [типа передачи](/docs/settings#documentsTransfer) добавили ввод значений свойств по умолчанию, которые будут подставляться при создании пакета (список оповещаемых) и формировании акта (отправитель, получатель, номер договора, название проекта):
![2026-13_-_настройки_передачи.png](/release_notes/2026-13_-_настройки_передачи.png =1200x)
- Добавили уведомление и лог при открытии комплекта ответственным за связанные замечания. Уведомление приходит администраторам проекта и пользователям, указанным на этапе `Оповещения` в настройках типа согласования. По умолчанию этот тип уведомления выключен, активировать его можно в профиле пользователя в [настройках уведомлений](/hub/user#notifications).

**:hammer_and_wrench: Исправление ошибок**
- Исправили сжатие изображения по высоте при вставке [факсимиле](/docs/folders#faximile).
- Поправили отображение компании в списке проектов в мобильной версии приложения.

> Добавили новые типы форм в констукторе и примеры шаблонов для генерации DOCX. Шаблоны размещены по ссылке: https://docs.sgnl.pro/s/f/019f6b6f-cdc7-71b0-920c-17b588e772d4
{.is-success}

**:sparkles: Доработки функционала**
- Сняли ограничение с администратора проекта на [редактирование завершенных форм](/forms/create).
- Добавили галочку `Уникальное значение в рамках проекта` для всех типов полей в [конструторе форм](/forms/builder). При активации пользователи не смогут сохранить форму со значением данного поля, если это значение уже используется в рамках данного типа формы проекта (например для ввода номера документа, ИНН компании, выборе записи ОЖР в АОСР).
- Добавили возможность отображать значение поля типа "Список вариантов" в имени формы (например для отображения типа и единиц измерения при выборе материала или вида работ).
- Сняли ограничение с количества вариантов у поля типа "Список вариантов": теперь можно оставить один вариант (например для обязательного подтверждения согласия в некоторых формах).
- Добавили предупреждение при закрытии формы, в которую внесли изменение и не сохранили.
- Добавили открытие форм из полей других форм в таблице.
- Добавили корзину для восстановления удаленных типов форм в [настройках проекта](/forms/settings).
- Разрешили использовать цифры в системных именах полей (системные имена используются в шаблонах для [генерации DOCX](/forms/generate)).
- Добавили логи изменений форм в организации HUB.

**:hammer_and_wrench: Исправление ошибок**
- Поправили сортировку отображаемых полей в имени формы.

**:fire: Карточка 18_Таблица из DOCS**

Новая карточка 18_Таблица из DOCS предназначена для отображения данных из Excel-файлов, загруженных в DOCS. При настройке карточки можно выбрать лист книги, указать диапазон отображаемых ячеек и настроить относительную ширину столбцов для оптимального отображения таблицы на превью карточки.
