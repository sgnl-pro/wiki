**:sparkles: Доработки функционала**
- В управление видимостью и положением столбцов в таблицах добавили ещё и пользовательские атрибуты:
![2026.2_-_управление_видимостью_атрибутов.png](/2026.2_-_управление_видимостью_атрибутов.png =800x)
- Добавили подсказки о ролях пользователей и о пользователях, к которым относится роль, в выпадающих списках:
![2026.2_-_пользователи_и_роли_в_тултипе.png](/release_notes/2026.2_-_пользователи_и_роли_в_тултипе.png =1200x)
- Добавили открытие элементов меню в новой вкладке при нажатии колёсиком или <kbd>Ctrl+</kbd>.
- В [историю версии файла](/docs/folders) добавили связь скопированной при согласовании версии файла с исходной версией, к которой можно перейти:
![2026.3_-_связь_скопированной_версии.png](/release_notes/2026.3_-_связь_скопированной_версии.png =800x)
- При создании [пометок](/docs/issues) теперь выбирается последний используемый инструмент.
- Добавили возможность пакетного импорта [комплектов согласований](/docs/reviews) из Excel для пользователей с правами на создание и редактирование (ранее была доступна только администраторам):
![2026.4_-_пакетный_импорт_согласований.png](/release_notes/2026.4_-_пакетный_импорт_согласований.png =800x)
- Вывели в комплект согласования максимально допустимую дату завершения проверки этапа:
![2026.2_-_дата_завершения_проверки.png](/release_notes/2026.2_-_дата_завершения_проверки.png =800x)
- Добавили в информацию при наведении на QR из PDF по результату согласования дополнительные свойства файла: проект, путь к файлу, пользовательские атрибуты и информацию о версии.
![2026.2_-_свойства_по_qr.png](/release_notes/2026.2_-_свойства_по_qr.png =600x)
- При [копировании типов](/docs/settings) замечаний и согласований из проекта в проект теперь отдельно показываем не найденные в шаблонном проекте типы:
![2026.2_-_копирование_настроек.png](/release_notes/2026.2_-_копирование_настроек.png =600x)
- В настройках типа согласования добавили необязательное поле для выбора пользователей или ролей, которые станут наблюдателями замечаний по умолчанию:
![2026.2_-_наблюдатели_по_умолчанию.png](/release_notes/2026.2_-_наблюдатели_по_умолчанию.png =600x)

**:hammer_and_wrench: Исправление ошибок**
- Расширили поле переименования файла в папке.
- Добавили возможность повторно запускать файл CAD в обработку, а не загружать новую версию при ошибке.
- Добавили игнорирование пробелов при создании атрибута типа "Список".
- Исправили формирование превью в плиточном режиме при загрузке большого количества файлов.
- Поправили выбор года в календаре атрибута.
- Вернули перелистывание страницы PDF при открытом замечании.
- Поправили экспорт столбца "Состояние" из реестра согласований.

**:fire: Карточка 52_2_3D цвета 2**

Реализовали новую карточку с возможностью просмотра трёх моделей: проектной, строительной и подложки:

![2026.2_-_карточка_52_2.png](/2026.2_-_карточка_52_2.png)

**:sparkles: Доработки функционала**
- Добавили доступы на уровне общих показателей, которые наследуются от доступов проекта, а также могут настраиваться самостоятельно через карандашик.
- Расширили доступы на уровне групп: доступ на редактирование всего проекта или группы позволяет добавлять карточки.
- В [карточке 67](/dash/cards-60#card-67) теперь отображаем жёлтый статус вместо зелёного, если комплект согласован с замечаниями.

**:hammer_and_wrench: Исправление ошибок**
- Поправили отображение диаграммы при перемещении [карточки 21](/dash/cards-20#card-21) в другую группу.
- Поправили отображение дат в [41 карточке](/dash/cards-40#card-41).

<span style="background: linear-gradient(45deg, #7AA6CF, #7D96AD); color: white; padding: 2px 6px; border-radius: 4px; font-size: 24px;">API</span>

В [публичный API](/general/api) добавили:
- фильтр по ItemVersionId в `POST /public/v1/issues`
- поле `completedBy` в `ReviewStage` и `ReviewSigning`
- метод `POST /public/v1/reviews/versions/item-versions` для получения всех версий файлов согласований
- поле `topVersion.reviewType.reviewGroupId` в `/public/v1/reviews` и метод `/public/v1/reviews/groups` для получения групп согласований
- поля `contractDate` и `contractDaysDiff` в `/public/v1/reviews` для получения договорной даты комплекта согласования
