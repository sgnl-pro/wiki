## Работа с документами
  
<details>
    <summary style="font-size: 16px; color: #0D47A1; background: #E3F2FD; border-radius: 7px; border: 1px solid #64B5F6; display: flex; gap: 10px">
        <span style="font-size: 20px;">03</span>
        <span style="flex-grow: 1;">Загрузка файлов</span>
    </summary>

Для загрузки файлов в SIGNAL DOCS достаточно перетащить их из проводника, либо в нужной папке нажать **➕ → Файл** и в открывшемся окне выбрать нужные файлы.

Чтобы загрузить новую версию уже существующего в DOCS файла:
- если наименование файлов совпадает — просто перетащить файл, либо добавить через ➕, как описано выше
- если наименование файлов различается — нажать ПКМ по файлу в СОД  → **Загрузить версию** и выбрать файл с любым именем в проводнике.

![загрузка_файлов.gif](/загрузка_файлов.gif)
  
</details>

<details>
    <summary style="font-size: 16px; color: #0D47A1; background: #E3F2FD; border-radius: 7px; border: 1px solid #64B5F6; display: flex; gap: 10px">
        <span style="font-size: 20px;">04</span>
        <span style="flex-grow: 1;">Версионность файлов</span>
    </summary>

Каждый файл хранит в себе историю версий. Чтобы посмотреть предыдущие версии, нажмите на число рядом с названием документа:
- если у вас есть права на редактирование, вы можете сделать ревизию файла (сделать актуальной версией одну из предыдущих)
- если у вас есть права на скачивание, вы можете скачать любую версию
 
![версионность_.gif](/faq/версионность_.gif)

</details>

        <span style="font-size: 20px;">06</span>
        <span style="flex-grow: 1;">Просмотр и сравнение PDF</span>
    </summary>

Для просмотра PDF документов в SIGNAL DOCS не нужно никакого дополнительного ПО — достаточно кликнуть по нужному файлу.  Инструменты разделены на несколько групп:
  
**Левый верхний угол — инструменты для работы с PDF**
  
![просмотр_pdf.png](/faq/просмотр_pdf.png)
- Постраничная навигация
- Вписать лист по размеру экрана
- Список замечаний, назначенных на этот документ
- Подписание документа ЭЦП
- Скрыть/отобразить пометки, оставленные в документе
- Перемещение пометок между листами

**Сверху по центру — инструменты просмотра**
  
![росмотр_pdf_2.png](/faq/росмотр_pdf_2.png)
- Навигация по листам
- Масштаб листа
- Поворот листа
- Сохранение поворота

**Сверху справа — дополнительные инструменты**
  
![росмотр_pdf_3.png](/faq/росмотр_pdf_3.png)
- Открыть в Autodesk Forge (альтернативный просмотрщик, с функцией линейки)
- Скачать документ без пометок
- Скачать документ со всеми пометками
- Инструмент **Сравнение**
----
Для сравнения документов между собой нажмите на соответствующий инструмент и выберите предыдущую версию файла или другой файл, загруженный в СОД. Результатом сравнения будут наложенные друг на друга чертежи, изменения будут подсвечиваться красным и синим цветом:

![сравнение_pdf.gif](/faq/сравнение_pdf.gif)

</details>
  
<details>
    <summary style="font-size: 16px; color: #0D47A1; background: #E3F2FD; border-radius: 7px; border: 1px solid #64B5F6; display: flex; gap: 10px">
        <span style="font-size: 20px;">07</span>
        <span style="flex-grow: 1;">Просмотр и сравнение САПР форматов</span>
    </summary>

Для просмотра BIM форматов (`.ifc`, `.rvt`, `.dwg`, `.nwd`) также не нужно никакого ПО на компьютере. Обработка файлов для просмотра происходит на серверах. Инструменты собраны в одной группе в нижней части экрана по центру:
- Перемещение по модели от первого лица (ходилка)
- Линейка
- Сечение модели
- Уровни модели (созданные в ПО)
- Чертежи и виды модели (созданные и опубликованные в ПО)
- Категории модели (созданные в ПО)
- Свойства элемента (вся атрибутивная информация элемента)
- Настройки
- Избранные свойства (выбранная в ПО атрибутивная информация элемента)
  
С САПР-форматами также доступен инструмент сравнения версий (правый верхний угол). Выберите предыдущую версию и дождитесь результатов сравнения. В отдельном окошке появится информация об удаленных, добавленных и измененных элементах:
  
![просмотр_сапр.gif](/faq/просмотр_сапр.gif)

</details>

## Работа с замечаниями
  
<details>
    <summary style="font-size: 16px; color: #0D47A1; background: #E3F2FD; border-radius: 7px; border: 1px solid #64B5F6; display: flex; gap: 10px">
        <span style="font-size: 20px;">08</span>
        <span style="flex-grow: 1;">Создание замечания</span>
    </summary>

С помощью замечаний в DOCS можно создать задание к любому типу файлов. Для создания замечания к файлу перейдите в него и в правой части экрана найдите значок ![image9.png](/faq/image9.png =25x)
  
Выберите преднастроенный администраторами тип замечания и заполните поля, после чего нажмите сохранить:
  
![создание_замечания.gif](/faq/создание_замечания.gif)

По всем событиям в карточке замечания (комментарии, смена ответственного, добавление вложений) вам и ответственному будут приходить уведомления.

Пометки в PDF и DWG файлах можно рисовать на листах, в RVT файлах можно прикреплять замечания к элементам модели. Для этого выберите нужные элементы через <kbd>Shift</kbd> и нажмите **Сохранить выбор:**
  
![создание_замечания_2.gif](/faq/создание_замечания_2.gif)

</details>
  
<details>
    <summary style="font-size: 16px; color: #0D47A1; background: #E3F2FD; border-radius: 7px; border: 1px solid #64B5F6; display: flex; gap: 10px">
        <span style="font-size: 20px;">09</span>
        <span style="flex-grow: 1;">Отработка замечания</span>
    </summary>

Как только на вас будет назначено замечание, на почту придёт уведомление. Нажав по номеру замечания, либо по названию файла вы сразу окажетесь в нужной карточке:
  
![отработка_замечания.png](/faq/отработка_замечания.png)

Ознакомьтесь с замечанием и в зависимости от вашего решения поставьте ему статус **В работе** или **Отклонено.** После отработки замечания и загрузки новой версии файла в DOCS поставьте статус **На проверке.** Для проверяющего же остаются статусы **Не принято** или **Закрыто.**

![статусы_замечаний.png](/faq/статусы_замечаний.png)

</details>
  
<details>
    <summary style="font-size: 16px; color: #0D47A1; background: #E3F2FD; border-radius: 7px; border: 1px solid #64B5F6; display: flex; gap: 10px">

        <span style="flex-grow: 1;">Поиск замечаний</span>
    </summary>

Для поиска замечаний перейдите в их реестр и воспользуйтесь быстрыми фильтрами:

|-------------|----------------------|

| <div style="text-align: center;"><span style="background: #ECF0F3; color: #44AEDE; padding: 6px 20px; border-radius: 20px; font-weight: 800; display: inline-block;">На мне</span></div> | Замечания, в которых вы указаны как ответственный |
| <div style="text-align: center;"><span style="background: #ECF0F3; color: #44AEDE; padding: 6px 20px; border-radius: 20px; font-weight: 800; display: inline-block;">Просрочено</span></div> | Замечания, у которых прошла дата устранения |
| <div style="text-align: center;"><span style="background: #ECF0F3; color: #44AEDE; padding: 6px 20px; border-radius: 20px; font-weight: 800; display: inline-block;">1</span></div> | Замечания, дата устранения которых заканчивается завтра

Также вы можете воспользоваться расширенными фильтрами в правом верхнем углу. Откроется панель с фильтрами по статусу, ответственным, названию файла, датам и другим свойствам. Например для фильтрации всех замечаний, назначенных на себя, можно выбрать фильтр по ответственному и указать свою учетную запись:
  
![фильтрация_замечаний.gif](/faq/фильтрация_замечаний.gif)
  
Для сброса фильтрации нажмите на крестик рядом со значком фильтра.
  
Реестр замечаний можно экспортировать в форматах PDF и XLSX для дополнительной фильтрации или передачи пользователям, не работающим в СОД. Для этого в реестре замечаний в левом верхнем углу нажмите ![image8.png](/faq/image8.png =25x) и выберите тип экспорта.
  
</details>

## Работа с согласованиями
  
<details>
    <summary style="font-size: 16px; color: #0D47A1; background: #E3F2FD; border-radius: 7px; border: 1px solid #64B5F6; display: flex; gap: 10px">
        <span style="font-size: 20px;">11</span>
        <span style="flex-grow: 1;">Инициация согласования</span>
    </summary>

Есть два основных способа запустить процесс согласования.
  
**Способ 1.** Найдите нужный комплект в реестре согласований, перейдите в него, нажмите ➕ и добавьте файлы, затем нажмите **Отправить:**
  
![создание_комплекта_2.gif](/faq/создание_комплекта_2.gif)

**Способ 2.** Прямо в папке выберите нужные документы и нажмите ![image7.png =25х](/faq/image7.png)
	Выберите нужный комплект из списка и нажмите **Отправить.** Рядом с файлами появится кнопка перехода в комплект:
  
![создание_комплекта_.gif](/faq/создание_комплекта_.gif)
  
Если нужный комплект согласования не доступен, обратитесь к администратору проекта.
</details>
   
<details>
    <summary style="font-size: 16px; color: #0D47A1; background: #E3F2FD; border-radius: 7px; border: 1px solid #64B5F6; display: flex; gap: 10px">
        <span style="font-size: 20px;">12</span>
        <span style="flex-grow: 1;">Проверка этапа согласования</span>
    </summary>

Как только наступит ваш этап согласования, на почту придет уведомление. Нажмте по ссылке, чтобы открыть комплект:

![проверка_комплекта.png](/faq/проверка_комплекта.png)

Далее откройте каждый файл комплекта, поставьте к нему замечания или утвердите без замечаний. После проверки всех файлов и проставления статусов нажмите **Завершить этап.** При необходимости напишите комментарий. Документация перейдет следующему проверяющему:

![проверка_комплекта.gif](/faq/проверка_комплекта.gif)
  
</details>
