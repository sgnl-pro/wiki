**Передача** — инструмент для юридически значимого обмена документацией между компаниями (заказчик, подрядчик, субподрядчик) с формированием акта приёма-передачи, подписанием его электронной подписью (ЭЦП) и контролем доступа к файлам.
  
  
  
  
  
  
  
  
  
  
  
  

# Вкладки{.tabset}
## 1. Основные понятия
###### 1. Основные понятия {#main_info}  
  
> **Пакет передачи** — набор файлов (документов, моделей, изображений), объединённых для передачи от одной стороны (отправителя) другой стороне (получателю). Пакет имеет шифр, название, статус, тип, а также связанный с ним акт приёма-передачи. В реестре передач каждый пакет отображается одной строкой.

> **Акт приёма-передачи** — официальный документ, формируемый автоматически при создании пакета (если это предусмотрено настройками типа). Акт содержит:
> - шифр передаваемого комплекта документации;
> - перечень файлов с их именами, размерами и хэш-суммами (алгоритм ГОСТ 2012-256);
> - подписи отправителя и получателя (если требуется ЭЦП).
> 
> Акт может быть подписан электронной подписью обеими сторонами и служит доказательством передачи.

> **Отправитель** — сторона (юридическое лицо или его представитель), которая формирует пакет, подписывает акт (если требуется) и отправляет его получателю. Отправитель указывает компанию-отправителя в настройках пакета.

> **Получатель** — сторона, которая получает пакет. Получатель может быть:
> - пользователем проекта (с учётной записью SIGNAL);
> - ролью проекта;
> - сторонним лицом без доступа к системе (если акт публикуется для внешнего подписания).
> 
> Получатель уведомляется о поступлении пакета, знакомится с актом, подписывает его (если требуется) и после этого получает доступ к скачиванию файлов (в зависимости от настроек типа).

> **Тип передачи** — набор настроек, определяющий поведение пакета:
> - требуется ли формирование акта приёма-передачи;
> - требуется ли подписание акта ЭЦП отправителем и получателем;
> - можно ли скачивать файлы до подписания акта получателем;
> - папка для сохранения актов;
> - получатель по умолчанию (пользователь, роль или сторонний email).
>
> Типы передач настраиваются администратором проекта.

> **Статус передачи** — состояние пакета, отображаемое в реестре. Основные статусы:
> - <span style="color: #B6BEC7; font-weight: bold">Черновик</span> — пакет создан, но ещё не отправлен получателю
> - <span style="color: #F2C94C; font-weight: bold">Ожидает подписания отправителя</span> — отправитель должен подписать акт (если требуется ЭЦП)
> - <span style="color: #F2C94C; font-weight: bold">Отправлен</span> — пакет отправлен получателю, ожидается подписание получателем (или сразу доступен для скачивания, если подписание не требуется)
> - <span style="color: #4FCDA6; font-weight: bold">Завершён</span> — акт подписан обеими сторонами (или подписание не требовалось), файлы доступны для скачивания
> - <span style="color: #FF4758; font-weight: bold">Отклонён</span> — получатель отклонил акт (файлы не переданы)

**Электронная подпись (ЭЦП)** — средство для подтверждения подлинности акта приёма-передачи. Для подписания требуются:
> - флешка (токен) с сертификатом ЭЦП;
> - установленный плагин CryptoPro;
> - установленные сертификаты электронной подписи в системе.
>
> Подписание доступно как для отправителя, так и для получателя. Подпись накладывается на сформированный PDF-акт.

> **Хэш-сумма файла** — уникальный цифровой отпечаток файла, вычисленный по алгоритму ГОСТ 2012-256. Указывается в акте для каждого передаваемого файла и позволяет проверить целостность и неизменность файла после передачи.

> **Публикация акта для стороннего подписания** — функция, позволяющая отправить акт на подписание лицу, у которого нет учётной записи в SIGNAL и доступа к проекту. Такой получатель получает ссылку на веб-страницу с актом, может просмотреть его, подписать (если у него есть ЭЦП) или отклонить. После подписания отправитель получает уведомление и может завершить передачу.

## 3. Подписание пакета отправителем
###### 3. Подписание пакета отправителем {#senderSigning}
  
Следующим этапом является подписание акта приема-передачи со стороны отправителя. Нажмите “Подписать” и выберите свою подпись. (Для того, чтобы ваша подпись появилась в выпадающем списке вам нужны: флешка с подписью, установленный плагин [_CryptoPro_](https://www.cryptopro.ru/) и установленные сертификаты электронной подписи) 

![](https://lh7-rt.googleusercontent.com/docsz/AD_4nXe9wYqdtzmAuXvnq1pkwedPjzvUVAm5HBgNv-Xi-zZ3-gU0BtwBW7k2KG3VldSpS6fOG8VZe4af7PnWZofI5fMYzOZh82wXbHRW2OLzXxuggfZba_bgwmKFXmdrXgNE3qE?key=0w72pDhc1Q-bjFjEPHaI_2Sz =70%x)

![](https://lh7-rt.googleusercontent.com/docsz/AD_4nXfxl_Qp67zJMU-tlqhTNdYYszFPYw7AiT9Fuij2ha33xUSlg1trEgdkVthFNlsq1R5mo3I3VPFdeAKHq-aDTbIqWmKZHoyxTE78dMXTLtSWZG6Vn1f4EoS8fKcfRQQ8EA?key=0w72pDhc1Q-bjFjEPHaI_2Sz =30%x)

В сформированном и подписанном акте указаны:

-   Шифр передаваемого комплекта документации
-   Имя каждого передаваемого файла.
-   Его хэш-сумма, сформированная с помощью [_алгоритма GOST 2012-256_](https://docs.cntd.ru/document/1200095035?marker=7D20K3)
-   Размер каждого передаваемого файла  
    ЭЦП располагается в нижней части акта

![](https://lh7-rt.googleusercontent.com/docsz/AD_4nXek6XvxSKCTnm-n-Afaz5j0-VOuJmHEL5Z-LRpPrZUitdS4BHFKJW7srdEHLpkDaT_phyqlp5tt9ya_OWJs0vWym8tTkzbyaap_ypDhe0rzAn7Bp2vC8_LWjt5GEFcoNws?key=0w72pDhc1Q-bjFjEPHaI_2Sz =50%x)

## 5. Подписание пакета получателем
###### 5. Подписание пакета получателем {#recipientSigning}
  
Отправителю придет уведомление на почту со ссылкой на отправленный акт передачи. Следующим шагом будет ознакомление с актом приема-передачи и его подписание ЭЦП получателя. На данном шаге получатель не может скачать файлы, пока не подпишет или не отклонит акт
