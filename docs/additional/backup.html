BACKUP

 

В SIGNAL DOCS возможно настроить резервное копирование файлов проекта на сервер компании. Для этого необходимо скачать клиент резервного копирования и заполнить файл настроек.

 

  1. Скачать клиент резервного копирования и распаковать его в папку резервной копии.

Внимание! Папка резервной копии должна располагаться на диске с достаточным объемом свободной памяти


 

 

  1. Заполнить файл настроек appsettings.json. Открыть его можно с помощью стандартного блокнота
     


 

  1. “YOUR_PROJECT_ID” необходимо заменить на ID проекта из DOCS. Для этого перейдите в нужный проект в DOCS и скопируйте его ID из адресной строки.
     


 

  1. Данные для полей “YOUR_CLIENT_ID” и “YOUR_CLIENT_SECRET” нужно получить  на SIGNAL HUB в разеделе “Интеграции”
     


 

  1. Нажмите “Добавить”, напишите название и выберите все методы в строке “Scopes”.


     
  2. После создания интеграции появится “YOUR_CLIENT_SECRET”, скопируйте его в файл настроек


 


 

  1. Выберите созданную интеграцию и нажмите   В открывшемся окне будет “YOUR_CLIENT_ID”, скопируйте его в файл настроек.


 

 

  1. Сохраните заполненный файл настроек и запустите клиент резервного копирования, файлы начнут скачиваться в созданную ранее папку.
