После того, как типы форм и полей созданы в конструкторе суперадмином, их нужно добавить в конкретный проект. Этим занимается администратор проекта.

# Вкладки{.tabset}
## Добавление типа формы в проект

Нажмите **+** и выберите нужные типы форм из конструктора:
  
![добавление_в_проект.png](/forms/добавление_в_проект.png)

После добавления вы можете смотреть список полей формы и свойства полей, для этого нажмите на значок информации:
  
![информация_о_поле.png](/forms/информация_о_поле.png)
 
## Копирование типа формы
  
Один и тот же тип формы может быть продублирован в рамках проекта для разных потоков документов:
- для разделения доступов между участниками проекта (например субподрядчиками, которые должны создавать и видеть каждый свой перечень форм);
- для логического разделения экземпяров форм (например форма журнала работ может быть использована для разных видов работ, но экземпляры должны отображаться в разных таблицах).
  
Для этого нажмите **Копировать** и введите суффикс. Для каждой копии можно по-разному настроить доступы и видимость полей. Например, для подрядчика «Субподряд-1» можно задать суффикс ` -СУБ-1`.
  
При копировании все настройки этапов (сроки, ответственные, наблюдатели) также копируются, но могут быть изменены независимо для каждой копии.
  
![настройки_проекта.png](/forms/настройки_проекта.png)

## Назначение прав доступа

Для каждой копии типа формы в проекте можно задать права на три действия:
- **Чтение** – просмотр существующих экземпляров форм
- **Создание** – возможность создать новый черновик формы и редактировать этот черновик до завершения
- **Редактирование** – возможность изменять существующие экземпляры форм других авторов

Доступы выдаются на роли пользователей.
  
## Настройка видимости полей

Для каждой копии типа формы в проекте можно скрывать необязательные поля, для этого нажмите на глазик.
  
Скрытое поле не отображается при заполнении и не может быть заполнено. Это полезно, если какой-то реквизит не требуется для данного проекта или подрядчика.
  
## Настройка этапов согласования
  
Если у типа формы в конструкторе настроены этапы, то у каждой добавленной в проект копии типа администратор может назначить ответственного для каждого этапа: пользователя, роль, автора формы или оставить по умолчанию "Назначить в процессе".

Также можно изменить срок этапа на конкретное количество дней, либо вернуть настройки по умолчанию, в т.ч. формулу расчета из конструктора.
