После того, как типы форм и полей созданы в конструкторе суперадмином, их нужно добавить в конкретный проект. Этим занимается администратор проекта.

## Добавление типа формы в проект

Выбираете один из существующих типов форм (FormType) и указываете суффикс – дополнительную метку для номеров экземпляров в рамках проекта (например, для филиала «Москва» можно задать суффикс -МСК).
Итоговый номер экземпляра будет формироваться как [префикс_формы]-[номер]-[суффикс_проекта].
Пример: АСР-42-МСК.

Один и тот же тип формы может быть добавлен в проект несколько раз с разными суффиксами (разные потоки документов).

## Настройка видимости полей в проекте

Для каждого добавленного типа формы можно переопределить видимость любого поля – скрыть его в рамках конкретного проекта. Это делается в настройках ProjectFormFieldType, где для поля указывается флаг hidden.
Скрытое поле не отображается при заполнении и не может быть заполнено. Полезно, если какой-то реквизит не требуется для данного проекта.

## Назначение прав доступа

Для каждого ProjectFormType можно задать права на три действия:

Чтение (Read) – просмотр существующих экземпляров форм.

Создание (Create) – возможность создать новый черновик формы.

Редактирование (Update) – возможность изменять существующие экземпляры (черновики или завершённые, в зависимости от статуса).

Права назначаются ролям пользователей в проекте (через roleId). Это позволяет детально управлять тем, кто может работать с документами.
