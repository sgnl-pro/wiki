В модуле FORMS реализована генерация готовых документов на основе заполненных форм. Это позволяет быстро создавать официальные документы (акты, протоколы, письма) без ручного копирования данных.

## Как это работает
  
Шаблон документа добавляется администратором проекта в формате DOCX в настройках типа формы. В места, куда должны подставляться данные из формы, вставляются специальные метки – системные имена полей (задаются в конструкторе форм), взятые в двойные фигурные скобки. Например, для поля `Наименование объекта` с системным именем `objectName` в шаблоне пишется `{{objectName}}`.

Генерация происходит по запросу пользователя из карточки завершённой формы. Система собирает значения всех полей формы, включая связанные записи (поля типа `Значение из источника`), подставляет их вместо соответствующих меток в шаблоне и формирует готовый DOCX-файл.

## Правила подстановки значений
  
Тип поля | Как подставляется в документ
:----- | :-----
**Строка** | Текст как есть. 
**Число** | Число с учётом единицы измерения (если задана). Например: `12.5 м³`.
**Да/Нет** | `Да` или `Нет` (локализовано).
**Дата и время** | В формате, установленном в системе (например: `ДД.ММ.ГГГГ` или `ДД.ММ.ГГГГ ЧЧ:ММ`).
**Диапазон дат** | `с ДД.ММ.ГГГГ по ДД.ММ.ГГГГ`
**Список вариантов** | Выбранное значение (или значения через запятую, если множественный выбор).
**Файл** | Имя файла.
**Значение из источника** | Если источником является другая форма: система «раскрывает» её – подставляет значения полей этой связанной формы, используя их системные имена. Например, если поле Подрядчик ссылается на форму «Контрагенты», то в шаблоне можно использовать {{Подрядчик.Наименование}}, {{Подрядчик.ИНН}} и т.д.
**Пользователь** | Отображается полное имя пользователя.
  
Поддержка множественных значений
Если поле допускает множественный выбор (например, несколько материалов), в шаблоне можно указать специальную конструкцию для перебора элементов. В текущей версии поддерживается простой перебор через запятую или список с маркерами (в зависимости от настроек шаблона). Администратор может использовать синтаксис {{#поле}}...{{/поле}} для циклов – это оговаривается при создании шаблона.

## Ограничения
  
Генерация доступна только для форм, у которых в проекте привязан шаблон документа (templateId). Без шаблона кнопка генерации не отображается.

Вложенные источники (форма в форме) обрабатываются на глубину до 5 уровней.

Если связанная форма не заполнена или удалена, в документ подставляется пустое значение (или текст «не указано»).

На данный момент доступен только формат DOCX. В ближайших планах – поддержка PDF и XML.

## Использование
Откройте заполненную форму (любой статус).

Нажмите «Сгенерировать документ» (иконка с листом или в меню действий).

Подождите несколько секунд – файл будет создан и автоматически загружен.

При необходимости откройте файл и проверьте корректность подстановки.

Совет: Рекомендуется завершать форму перед генерацией, чтобы данные не изменялись в процессе создания документа. Однако генерация возможна и для черновиков – например, для промежуточного отчёта.

Для администраторов: Настройка шаблона выполняется в разделе «Настройки проекта» → «Управление шаблонами». Подробная инструкция по созданию шаблонов доступна в отдельной справке.
