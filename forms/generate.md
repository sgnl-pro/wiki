В модуле реализована генерация готовых документов на основе заполненных форм. Это позволяет быстро создавать официальные документы (акты, протоколы, письма) без ручного копирования данных.

# Вкладки{.tabset}
## 1. Как это работает
  
- Шаблон документа добавляется администратором проекта в формате DOCX в настройках типа формы. В места, куда должны подставляться данные из формы, вставляются специальные ключи – системные имена полей (задаются в конструкторе форм), взятые в двойные фигурные скобки. Например, для поля `Наименование объекта` с системным именем `objectName` в шаблоне пишется `{{objectName}}`.

- Генерация происходит по запросу пользователя из карточки завершённой формы. Система собирает значения всех полей формы, включая связанные записи (поля типа `Значение из источника`), подставляет их вместо соответствующих меток в шаблоне и формирует готовый DOCX-файл.
  
- Если в шаблоне содержатся неизвестные ключи (которых нет в форме), система выдаёт предупреждение с их списком, но генерация файла всё равно доступна.
  
Файлы шаблоны для генерации документов представлены по ссылке:
https://docs.sgnl.pro/s/f/019f6b6f-cdc7-71b0-920c-17b588e772d4
  
Вы можете настроить шаблоны под требования Вашей Компании и загрузить их в вашем проекте.


## 2. Правила подстановки значений
  
Тип поля | Как подставляется в документ
:----- | :-----
**Строка** | Текст как есть. 
**Число** | Число с учётом единицы измерения (если задана). Например: `12.5 м³`.
**Да/Нет** | `Да` или `Нет` (локализовано).
**Дата и время** | В формате, установленном в системе (например: `ДД.ММ.ГГГГ` или `ДД.ММ.ГГГГ ЧЧ:ММ`).
**Диапазон дат** | `с ДД.ММ.ГГГГ по ДД.ММ.ГГГГ`
**Список вариантов** | Выбранное значение (или значения через запятую, если множественный выбор).
**Файл** | Для графических файлов вставляются имя файла и само изображение, для остальных только имя.
**Значение из источника** | Если источником является другая форма: система «раскрывает» её – подставляет значения полей этой связанной формы, используя их системные имена. Например, если поле `Подрядчик` с системным именем `contractor` ссылается на форму `Контрагенты`, то в шаблоне можно использовать `{{contractor.name}}`, `{{contractor.address}}` и т.д. При множественном выборе значения перечисляются через запятую. Поддерживаются связи глубиной до пятого уровня.

## 3. Ограничения текущей версии
- Генерация доступна только для форм, у которых в настройках проекта добавлен шаблон документа. Без шаблона кнопка генерации не отображается.
- Вложенные источники (форма в форме) обрабатываются на глубину до 5 уровней.
- Если связанная форма не заполнена или удалена, в документ подставляется пустое значение.
- Элементы списков при множественном выборе перечисляются через запятую.
- Доступен только формат DOCX.
