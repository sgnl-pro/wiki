Пользователи с соответствующими правами создают и заполняют экземпляры форм (собственно формы). Форма всегда привязана к конкретному проекту и типу формы.

# Вкладки{.tabset}
## 1. Поиск и просмотр форм
  
Выберите тип формы из доступных в выпадающем списке. Для каждого типа формы в проекте доступен реестр (таблица) со списком созданных форм.
  
Колонки таблицы – это поля, которые специальным образом отмечены в конструкторе, а также системные свойства формы: номер, тип в проекте, автор, последний редактор, дата создания, дата последнего изменения, текущий этап, его ответственный и плановая дата завершения.
  
![выбор_формы_в_реестре.png](/forms/выбор_формы_в_реестре.png)
  
В реестре вы можете:
- искать по номерам, названиям форм, датам, ответственным, 
- сортировать по номеру, дате создания, дате изменения, типу формы
- открывать форму для просмотра или редактирования (если она в черновике и у вас есть права)
- переходить в связанные формы, указанные в качестве значений полей, и возвращаться обратно
  
## 2. Создание новой формы

В нужном реестре нажмите **+ Добавить.** Если у вас есть права на создание более чем одного дубликата данного типа формы, то выберите один из них, ориентируясь на суффиксы.
  
![выбор_формы_проекта.png](/forms/выбор_формы_проекта.png)

Система автоматически присвоит номер формы (следующий по порядку для данного типа формы проекта).

Форма создаётся в статусе **«Черновик»**. В черновике можно сохранять любые значения, даже если обязательные поля не заполнены. Если для формы настроен маршрут согласования, после запуска процесса она переходит в статус «На согласовании». Успешно прошедшая все этапы форма получает статус «Завершена».

## 3. Заполнение полей

При заполнении вы видите только те поля, которые не скрыты в настройках проекта и для которых выполнены условия видимости:
  
![карточка_формы.png](/forms/карточка_формы.png)

Типы полей отображаются соответствующими элементами управления:
- текстовое поле с возможной маской (например, для телефона)
- поле ввода числа с указанием единицы измерения и контролем допустимых диапазонов
- переключатель (да/нет)
- календарь, в котором может быть только дата или дата + время
- два календаря (начало, конец)
- список вариантов с одиночным или множественным выбором
- загрузка одного или нескольких файлов
- выбор одной или нескольких заполненных форм других типов (например, выбрать форму `Документ, подтверждающий качество` в качестве значения формы `Запись журнала входного контроля материалов`). 

Если в настройках поля с выбором заполненных форм используются фильтры, то список доступных для выбора форм автоматически ограничивается. Например, поле `Примененные материалы` может показывать только материалы, у которых `Тип материала` совпадает с выбранным в текущей форме типом.
  
## 4. Согласование форм
  
Для форм, у которых в конструкторе настроен маршрут согласования, происходит процесс поэтапного заполнения и утверждения.
  
![этапы_в_реестре_форм.png](/forms/этапы_в_реестре_форм.png)
  
### Запуск согласования
  
После заполнения всех обязательных полей черновика автор может отправить форму на первый этап.
  
Система предложит указать ответственного для следующего этапа (или нескольких, если этапы параллельны), если ответственный не задан в настройках проекта, и ввести комментарий (необязательно).

После подтверждения форма переходит в статус «На согласовании», и ответственные получают уведомления.

### Прохождение этапов
  
В карточке формы отображаются:
- статус формы и название
- цепочка с завершенными и текущим этапами маршрута
- ответственного за текущий этап
- плановую дату завершения текущего этапа (рассчитывается автоматически)
- поля предыдущих этапов, недоступные для редактирования
- поля текущего этапа (если есть), доступные ответственному для редактирования
- кнопки действий
  
**Действия для просмотра**
- при наведении на этап в цепочке появляется подробная информация о нём
- при нажатии на этап форма листается до его полей (если они есть и если не отображаются на экране)
- для копирования информации об этапе в буфер обмена нажмите на него правой кнопкой мыши

**Действия ответственного**
- **Сохранить черновик** – ответственный может заполнить часть полей, но не передавать форму на следующий этап.
- **Завершить этап** – ответственный оставляет комментарий (необязательно) и, при необходимости, назначает ответственных для следующих этапов. После завершения всех активных этапов (включая параллельные) процесс переходит к следующим.
- **Отклонить этап** – ответственный возвращает этап на доработку (требуется комментарий). Форма переходит на предыдущий этап (или возвращается в черновик, если это был первый этап).
  
**Действия администратора и редактора**
- Изменение ответственного текущего этапа.
- Изменение плановой даты текущего этапа.
- **Отменить согласование** – редактор или администратор может полностью остановить процесс (требуется комментарий). Форма возвращается в черновик.

### Параллельные этапы

Если в маршруте есть параллельные этапы, несколько этапов могут быть активны одновременно. Каждый ответственный работает со своей частью полей независимо. Система автоматически завершает группу, когда все параллельные этапы завершены.

### Пример сценария

1. **Автор** создаёт форму «Акт скрытых работ», заполняет общую информацию (место, даты, материалы) и нажимает «Передать на согласование».
2. **Инженер технадзора** получает уведомление, открывает форму – ему доступны поля этапа «Проверка». Он вносит заключение и завершает свой этап.
3. **Главный инженер** видит следующий этап «Утверждение», проверяет данные и завершает его. Форма автоматически переходит в статус «Завершена».
4. Если на каком-то этапе обнаружены ошибки, ответственный нажимает «Отклонить этап» – форма возвращается автору или ответственному за предыдущий этап с комментарием. После исправления согласование запускается повторно

## 5. Завершение формы
  
Для форм без маршрута согласования кнопка «Завершить» доступна сразу после заполнения обязательных полей. Для форм с этапами завершение происходит автоматически после прохождения всех этапов.

После завершения форму нельзя редактировать и можно использовать в качестве значения полей других форм:
  
![завершение_формы_2.png](/forms/завершение_формы_2.png)

## 6. Удаление и восстановление формы
  
Формы могут удалять и восстанавливать из корзины только редакторы и администраторы проекта. Восстановление возвращает форму в статус, который был до удаления.
  
![удаление_форм.png](/forms/удаление_форм.png)
  
![восстановление_форм.png](/forms/восстановление_форм.png)
  
## 7. Уведомления

При каждом значимом событии (запуск согласования, начало этапа, завершение этапа, возврат) ответственные и наблюдатели получают уведомления на email или в мессенджер.
  
Настройка уведомлений выполняется в [профиле пользователя](/hub/user#notifications).
