Пользователи с соответствующими правами создают и заполняют экземпляры форм (ProjectForm). Экземпляр всегда привязан к конкретному проекту и к одному ProjectFormType.

## Создание новой формы
Выберите тип формы из доступных (те, на которые у вас есть право Create).

Система автоматически присвоит номер (следующий по порядку для данного ProjectFormType).

Экземпляр создаётся в статусе «Черновик» (Draft). В черновике можно сохранять любые значения, даже если обязательные поля не заполнены.

## Заполнение полей
Пользователь видит только те поля, которые не скрыты в настройках проекта и для которых выполнены условия видимости.

Типы полей отображаются соответствующими элементами управления:

String – текстовое поле с возможной маской (например, для телефона).

Numeric – поле ввода числа с указанием единицы измерения и контролем допустимого диапазона/шага.

Boolean – флажок (чекбокс).

DateTime – календарь, может быть только дата или дата+время.

DateRange – два календаря (начало, конец).

Selection – выпадающий список (одиночный или множественный).

File – загрузка файлов (можно прикрепить несколько).

FormSource – выбор записи из другой формы (например, выбрать контрагента из справочника). При выборе можно использовать поиск и фильтры, заданные конструктором.

Примечание про FormSource: Если в настройках поля указаны фильтры (равенство полей), то список доступных для выбора записей автоматически ограничивается. Например, поле «Применённые материалы» может показывать только материалы, у которых «Тип материала» совпадает с выбранным в текущей форме типом.

## Завершение формы
Когда все обязательные поля заполнены, пользователь может перевести форму в статус «Завершена» (Completed). После этого:

Форму нельзя редактировать (если только у пользователя нет специальных прав, но по умолчанию завершённые формы защищены).

Форма появляется в реестре с возможностью просмотра (для тех, у кого есть право Read).

Некоторые поля могут использоваться в других формах как источник данных (FormSource).

Восстановление из корзины: Если форма была удалена (мягкое удаление), её можно восстановить через функцию «Восстановить». Это возвращает форму в статус, который был до удаления.

## Поиск и реестр форм
Для каждого типа формы доступен реестр (таблица) со списком экземпляров.
Колонки таблицы – это поля, у которых включён флаг «Использовать как колонку реестра». В реестре можно:

Искать по номеру, текстовым полям, датам, ответственным.

Сортировать по номеру, дате создания, дате изменения, типу формы.

Открыть форму для просмотра или редактирования (если она в черновике и есть право Update).
