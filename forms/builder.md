В этом разделе администратор или ответственный пользователь создаёт типы форм (FormType) и типы полей (FormFieldType). Это библиотека шаблонов, которые потом можно будет использовать в конкретных проектах.

1.1. Типы форм (FormType)
Каждый тип формы задаёт структуру документа (например, «Акт освидетельствования», «Журнал бетонных работ»). При создании указывается:

Название – отображаемое имя (например, Акт скрытых работ).

Префикс – короткая буквенная метка для нумерации (например, АСР). Она будет автоматически добавляться к номеру экземпляра формы.

Описание (необязательно).

Порядок – для сортировки в списке типов форм.

Группа – принадлежность к группе типов (FormTypeGroup), если используется.

Созданный тип формы ещё не привязан к проекту – он является общим шаблоном.

1.2. Типы полей (FormFieldType)
Для каждого типа формы можно создать несколько полей. Доступно 8 видов полей:

Вид поля	Назначение	Основные настройки
String	Текстовая строка	Маска ввода, единица измерения, значение по умолчанию, валидация Regex
Numeric	Число	Единица измерения, значение по умолчанию, валидация (диапазон, шаг)
Boolean	Да/Нет	Значение по умолчанию (включено/выключено)
DateTime	Дата и/или время	Режим (только дата или дата+время), значение по умолчанию (фиксированное или текущее ± дни)
DateRange	Интервал дат	(без дополнительных настроек, задаётся начало и конец)
Selection	Выбор из списка	Множественный выбор, варианты ответов (не более 20), выбор по умолчанию
File	Загрузка файлов	Множественная загрузка
FormSource	Выбор записи из другой формы	Множественный выбор, связь с формой-источником, фильтрация по равенству полей
Общие свойства любого поля:

Название – подпись поля в форме.

Описание – вспомогательный текст (отображается под полем).

Системное имя – уникальный технический идентификатор (если нужно).

Порядок – очередность отображения.

Использовать как отображаемое значение – если поле выбрано, его значение будет показываться в списках (реестрах) рядом с номером формы.

Использовать как колонку реестра – поле появляется в таблице со списком экземпляров форм.

Обязательное поле – требует заполнения перед завершением (переводом в статус «Завершена»).

Условие видимости – поле отображается только если выполнено логическое условие, основанное на значениях других полей этой же формы (например, поле «Причина отказа» показывать только при выборе «Отказано»).

Важно: Условия видимости могут использовать константы (= Да, = 100), сравнение двух полей (Дата начала <= Дата окончания) и группировку с И/ИЛИ. Они не влияют на обязательность: если поле скрыто, оно не требуется для заполнения.
