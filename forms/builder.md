В этом разделе суперадмин создаёт типы форм и их полей. Формируется библиотека шаблонов, которые потом можно будет использовать в конкретных проектах.

## Типы форм
Каждый тип формы задаёт структуру документа (например, «Акт освидетельствования», «Журнал бетонных работ»). При создании указывается:

1. **Группа форма** (группы создаются здесь же в конструкторе. например `Акты и протоколы`)

1. **Название** – отображаемое имя (например `Акт скрытых работ`)

2. **Префикс** – короткая буквенная метка для идентификации (например, `АОСР`). Она будет автоматически добавляться к номеру экземпляра формы в проекте

3. **Описание** (необязательно)

4. **Порядок** – группы и формы можно перемещать для правильной сортировки

## Типы полей
В каждом типе формы можно создавать список полей. Доступно 8 видов полей:

Текстовая строка	Маска ввода, единица измерения, значение по умолчанию, валидация Regex

Число	Единица измерения, значение по умолчанию, валидация (диапазон, шаг)

Да/Нет	Значение по умолчанию (включено/выключено)

Дата и/или время	Режим (только дата или дата+время), значение по умолчанию (фиксированное или текущее ± дни)

Интервал дат	(без дополнительных настроек, задаётся начало и конец)

Выбор из списка	Множественный выбор, варианты ответов (не более 20), выбор по умолчанию

Загрузка файлов	Множественная загрузка

Выбор записи из другой формы	Множественный выбор, связь с формой-источником, фильтрация по равенству полей

Общие свойства любого поля:

Название – подпись поля в форме.

Описание – вспомогательный текст (отображается под полем).

Системное имя – уникальный технический идентификатор (если нужно).

Порядок – очередность отображения.

Использовать как отображаемое значение – если поле выбрано, его значение будет показываться в списках (реестрах) рядом с номером формы.

Использовать как колонку реестра – поле появляется в таблице со списком экземпляров форм.

Обязательное поле – требует заполнения перед завершением (переводом в статус «Завершена»).

Условие видимости – поле отображается только если выполнено логическое условие, основанное на значениях других полей этой же формы (например, поле «Причина отказа» показывать только при выборе «Отказано»).

Важно: Условия видимости могут использовать константы (= Да, = 100), сравнение двух полей (Дата начала <= Дата окончания) и группировку с И/ИЛИ. Они не влияют на обязательность: если поле скрыто, оно не требуется для заполнения.
